Claude Code 常见误区

每个新手都会犯。提前知道，少走弯路。

❌ 一次性让 AI 写完整项目
✅ 小步迭代：先跑通核心功能，再逐步加模块
为什么？ AI 上下文窗口有限（~200K tokens），一次性塞满会导致逻辑混乱。拆成 3-5 步效果最好。
❌ 不写需求文档直接开始
✅ PRD 驱动：先让 AI 写需求文档，review 后再编码
为什么？ 没有验收标准，AI 会往错的方向狂写。200 字需求文档能省 2 小时返工。
❌ AI 产出直接合并，不做审查
✅ 每次生成后 review + 测试，确认无安全漏洞再合并
为什么？ AI 会写出逻辑正确但安全有漏洞的代码。Review 是最后一道防线。
❌ 不做版本控制
✅ 每次 AI 操作前 git commit，出问题直接 git reset
为什么？ AI 可能误删关键文件。养成肌肉记忆：AI 操作 → diff → commit。
