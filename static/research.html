📖 开源贡献完整流程

从 Fork 到 Merge，每一步都有清晰指引

1

Fork 仓库

在 GitHub 上点击 Fork，将项目复制到你的账户下

2

Clone 到本地

git clone 你的Fork仓库，在本地进行开发

3

创建分支

git checkout -b feature-name，在独立分支上工作

4

编码 & 提交

编写代码，遵循项目规范，写好 commit message

5

Push & PR

推送到你的Fork，在 GitHub 上创建 Pull Request

6

Code Review

维护者审查代码，根据反馈修改，直到合并

📋 高质量 PR 检查清单

✅ 阅读 CONTRIBUTING.md
✅ 单个 PR 只解决一个问题
✅ 遵循项目的代码风格
✅ 添加必要的测试用例
✅ 更新相关文档
✅ 写清楚 PR 描述（What/Why/How）
✅ 关联相关 Issue（Fixes #123）
✅ 耐心等待 review，友好回应
