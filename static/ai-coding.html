常见误区

每个 AI 编程新手都会犯，提前知道，少走弯路

✕ 一次性让 AI 写完整项目AI 上下文窗口有限，一次性塞满会导致逻辑混乱。
✓ 小步迭代先跑通核心功能，再逐步加模块。拆成 3-5 步效果最好。
✕ 不写需求文档直接开始没有验收标准，AI 会往错的方向狂写。
✓ PRD 驱动先让 AI 写需求文档，review 后再编码。200 字需求能省 2 小时返工。
✕ AI 产出直接合并，不做审查AI 会写出逻辑正确但安全有漏洞的代码。
✓ 审查后再合并每次生成后 review + 测试，确认无安全漏洞再合并。
✕ 不做版本控制AI 可能误删关键文件，没有后悔药。
✓ AI 操作前 git commit养成肌肉记忆：AI 操作 → diff → commit，出问题直接 git reset。
